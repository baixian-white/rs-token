# PPT 图片生成提示词集

> 风格基调：**学术简洁 + 强视觉锚点**。白底 / 浅灰底，深蓝主色 + 单一暖橙强调色，思源黑体，几何直线，硬边框方正排版。Distill.pub / Anthropic 论文图美学。生成工具：GPT image2 pro。
>
> **整套 PPT 的视觉语言（每页都遵守）**：
> - **深蓝(#1F3A68)** 代表"结构 / 全局 / 通用机制"
> - **暖橙(#E07B3C)** 仅用于"语义 / 关键 / 降级故事线" —— 眼睛沿着橙色就能扫完整张图的论点
> - 强调色**绝不撒胡椒面**,一页里出现的橙色必须共属一条逻辑线

## Page 1 — SpeechTokenizer 的真正贡献

### 页面目标
让导师在 30 秒内 get 到：SpeechTokenizer 的创新点不是 RVQ 本身，而是**用 HuBERT 蒸馏让 RVQ 第一层承载"音素语义"**。

### 设计意图（向 image2 pro 解释 why，不是直接喂入）
- **整张幻灯片的几何骨架是一条横贯整页的 RVQ 流水线**。所有其他模块（核心论断、降级传输）都通过细线"挂载"在这条流水线上,骨架本身就是页面的记忆点
- 流水线里 L1 矩形宽度是后三层的 2.5 倍,且是全页唯一的橙色 —— **一个骨架 + 一个焦点**
- 核心论断卡片挂在 L1 正下方 —— 视觉上直接说"我们的论断对象就是 L1"
- 降级传输载荷对比的色块比例**严格复用**流水线的几何比例(L1 宽 vs L2-L4 各窄) —— 不同模块共享同一套几何语言,加深"骨架感"
- 把语义分工表融进流水线下方的标注 —— 流水线本身就是表,省一个独立模块,让骨架更纯粹

## Page 2 — 已有工作排查 (上)：检索方法论与高相关候选

### 页面目标
向导师呈现**研究空隙的论证不依赖直觉,而是基于系统性文献检索 + 对每篇高相关论文的细读差异化**。本页回答两个问题:检索覆盖了哪些角度?最相关的若干篇为何不构成 prior work?

### 设计意图
- 视觉骨架沿用 Page 1 的"脊柱 + 挂载"思路 —— 此处脊柱为一条**纵向检索流程线**(上至下:四个检索角度 → 漏斗筛选 → 高相关候选)
- 五篇候选论文采用**统一卡片网格**,每张卡片右下角有明确的差异化标签,视觉上形成整齐对照阵列
- 唯一橙色焦点:MOC-RVQ 卡片(因技术路线最接近,但仍在应用域与机制上差异化)
- 全图不使用 stock 图标,差异化标签通过纯文字符号 ✓ ✗ 表达
